Intentional auth split. Web owns its backend (Better Auth + DB). SPA and Expo hit an external public DummyJSON auth API so client apps can be exercised without spinning up a database.

Intentional UI mirror. packages/core must run in all three apps including Expo. SPA and Web UI modules use React Aria (web-only) — they stay in each web app, not in core. The ~89 mirrored components are deliberate, not drift waiting to merge into core.

Top recommendation

Deepen Client Session guards — real bugs (duplicate guards, Bearer header on the wrong request) live inside spa/expo client session handling, independent of the intentional web vs DummyJSON auth split.

Close second: Deepen Http into ApiClient — header side-effects leak inside repositories; pairs with DummyJSON client auth.

Documented intent, not merges: auth split (#1), spa/web UI mirror (#3).
